## Overview

This registry records implementations and adopters of the x402 STARK Receipts
canonical specification (`draft-vauban-x402-stark-receipts` and companion drafts).
It is maintained by Vauban Pay as primary author of the specification track.

<Note>
  The registry is informational and reflects reported, verified adoptions only.
  Omission does not imply non-conformance.
  To register an adoption, see the [Adoption Process](#adoption-process) section.
</Note>

**Canonical version pin** : `jcs-rfc8785-v1`
Accepted alternate forms: `x402-jcs-v1.0.0` and `1.0`
Canonicalisation baseline: [RFC 8785](https://www.rfc-editor.org/rfc/rfc8785) (JSON Canonicalization Scheme)

**Registry version** : 0.1.0 ; last updated 2026-05-25

## Adoption Categories

<CardGroup cols={1}>
  <Card title="Category 1 : Production Adopter" icon="rocket">
    The organisation runs the x402 STARK Receipts spec in live payment flows, emitting
    `PAYMENT-SIGNATURE` or `PAYMENT-RESPONSE` headers that conform to the `jcs-rfc8785-v1`
    canonicalisation discipline.

    **Required disclosures at registration :**
    - Production endpoint (may be anonymised domain)
    - `canon_version` emitted
    - JCS library used and version
    - IETF I-D revision adopted
  </Card>

  <Card title="Category 2 : Conformance Implementer" icon="code">
    The organisation has published a byte-for-byte-validated runner for at least one
    conformance vector set in this repository, in any language, producing 8/8 agreement
    across all vectors in that set.

    **Required disclosures at registration :**
    - Language and JCS library used
    - Vector set(s) validated
    - Runner location (source file URL or package reference)
    - Result count (e.g. `8/8` per vector set)
  </Card>

  <Card title="Category 3 : Reference Citation" icon="book">
    The organisation cites one or more drafts from the `draft-vauban-x402-*` IETF I-D
    family in their own normative documentation, specification, or Internet-Draft.

    **Required disclosures at registration :**
    - Document title and URL citing the I-D
    - Which I-D(s) are cited
    - Citation context (normative / informative)
  </Card>
</CardGroup>

## Adoption Process

<Steps>
  <Step title="Prepare disclosures">
    Gather the required disclosures for your adoption category (Production Adopter,
    Conformance Implementer, or Reference Citation). See category definitions above.
  </Step>

  <Step title="Send registration email">
    Email `research@vauban.tech` with subject line `[x402-stark-receipts] Adoption registration`.
    Include your organisation name, contact email, category, and required disclosures.
    Consent to public listing under Apache-2.0 terms is required.
  </Step>

  <Step title="Verification">
    Verification covers two checks:

    1. **Canonical vector reproduction** : your reported `canon_version` and JCS library
       reproduce at least one full vector set with 8/8 byte-identical digests.

    2. **Cross-axis interop binding check** (Production Adopters only) : a live
       `PAYMENT-REQUIRED` challenge is issued; the returned `PAYMENT-SIGNATURE` header
       is parsed against the wire format in `draft-vauban-x402-stark-receipts-01` Section 4.

    Verification is completed within 10 business days.
  </Step>

  <Step title="Listing">
    Verified adopters are listed in this registry and in `ADOPTERS.md` at the
    conformance vectors repository root. Listing includes a `verified` status marker.
  </Step>
</Steps>
